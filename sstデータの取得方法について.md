# 衛星データ（GCOM-C・ひまわり）の取得方法とスペック詳細

## 1. データ取得プロトコル

私が使用する各衛星データの提供プラットフォームと，具体的なアクセス手順を以下にまとめます．

### GCOM-C (しきさい / SGLI)
JAXAが運用する地球観測衛星．沿岸部での250mという極めて高い空間解像度が特徴です．

* **プラットフォーム**: [JAXA G-Portal](https://gportal.jaxa.jp/gpr/)
* **プロダクトファイル名**:
    * gcom-cのファイル名はグラニュールIDにHDF5を示す拡張子「.h5」を付与したものとなっている．
    * GCOM-Cのファイル名は計41バイト（文字）で構成されており，大きくSceneID（観測情報）とProductID（処理情報）の2つのセクションに分かれています．
    #### 1\. SceneID（1〜20バイト：観測に関する識別子）

    | バイト位置 | 項目名 | 内容・設定値の詳細 |
    | :--- | :--- | :--- |
    | **1 - 3** | Satellite | 衛星名．`GC1` (GCOM-C1) で固定． |
    | **4 - 6** | Sensor | センサ名．`SG1` (SGLI) で固定． |
    | **7** | (セパレータ) | `_` (アンダースコア) |
    | **8 - 11** | Year | 観測開始時の西暦 (UTC) |
    | **12 - 13** | Month | 観測開始時の月 (UTC) |
    | **14 - 15** | Day | 観測開始時の日 (UTC) |
    | **16 - 17** | Hour | 観測開始時の時 (UTC) |
    | **18 - 19** | min | 観測開始時の分 (UTC) |
    | **20** | sec | **秒数を示す1桁のアルファベット**（※注1参照） |

    #### ※注1：秒数アルファベット対応表

    観測開始ラインの秒数を3秒刻みで1文字のアルファベット（I， Oを除く）で表記します．

    * **A**: $0 \le s < 3$ / **B**: $3 \le s < 6$ / **C**: $6 \le s < 9$ ...
    * **L**: $30 \le s < 33$ / **P**: $39 \le s < 42$ ...
    * **W**: $60 \le s < 61$ (うるう秒)

    -----

    #### 2\. ProductID（21〜41バイト：プロダクトに関する識別子）

    | バイト位置 | 項目名 | 内容・設定値の詳細 |
    | :--- | :--- | :--- |
    | **21 - 23** | Path | パス番号（軌道番号）．設定値：`001`〜`485` |
    | **24 - 25** | Scene | シーン番号．設定値：`01`〜`24` |
    | **26** | (セパレータ) | `_` (アンダースコア) |
    | **27 - 28** | Level | 処理レベル．L2プロダクトの場合は `L2` |
    | **29** | Type (x1) | プロダクト種別．`S` (標準プロダクト) で固定． |
    | **30** | Type (x2) | 運用処理種別．<br>・`G`: 全球処理<br>・**`L`: 準リアルタイム処理 (特定地域)**<br>・`N`: 準リアルタイム処理 (全球) |
    | **31** | (セパレータ) | `_` (アンダースコア) |
    | **32 - 35** | Subsystem | プロダクトID（物理量）．SSTの場合は `NWLR` (水色) や `SST_` 等が入る． |
    | **36** | Resolution (r) | 空間解像度．<br>・`K`: 1000m / `H`: 500m / **`Q`: 250m** |
    | **37** | (セパレータ) | `_` (アンダースコア) |
    | **38** | algorithm ver. | アルゴリズムバージョン．設定値：`0`〜`9`, `A`〜`Z` |
    | **39 - 41** | parameter ver. | パラメータバージョン．設定値：`000`〜`999` |
    -----
    * 例として次のファイル名`GC1SG1_202105101239L28803_L2SG_SSTNQ_2000.h5` を，先ほどのグラニュールIDの命名規則に基づいて読み解く場合を以下に記します．
    #### 1. SceneID（観測情報）
    * **GC1SG1**: 衛星「GCOM-C1」，センサ「SGLI」
    * **202105101239L**: 観測日時（UTC）
        * 日付: **2021年5月10日**
        * 時刻: **12時39分**
        * 秒数: **33秒 〜 36秒の間**（アルファベット **`L`** については※注1：秒数アルファベット対応表を参照）
        * *JST（日本時間）では 2021年5月10日 21時39分台になります．*

    #### 2. ProductID（処理情報）
    * **28803**: パスとシーン
        * パス番号: **288**
        * シーン番号: **03**
    * **L2SG**: 処理レベル
        * レベル: **L2**
        * プロダクト種別（x1）: **S**（標準プロダクト固定）
        * 運用処理種別（x2）: **G**（**全球処理**）
    * **SSTNQ**: サブシステムと解像度
        * プロダクトID: **SSTN**（海面水温：準リアルタイム用プロダクトコード）
        * 解像度（r）: **Q**（**250m解像度**）
    * **2000**: バージョン
        * アルゴリズムVer: **2**
        * パラメータVer: **000**
    ＊上記のように読み解けるようになると欲しいファイルを探す時に役立ちます．

* **取得データ**: L2 SST 準リアルタイム (Near Real-Time)
* **取得手順1**:
    1.  **アカウント作成**: G-Portalにてユーザー登録を行い，SFTPアクセス用のパスワードを設定．
    2.  **プロダクト選択**: センサー ”SGLI” → レベル ”L2” → 物理量 ”SST” を指定．
    3.  **時間・領域指定**: 解析対象期間と日本近海の座標範囲を入力し，対象ファイルを検索・注文（Order）．
    4.  **データ取得**: 注文完了後，**SFTPクライアント**（FileZilla/cyberduck等）を用いて ”sftp.gportal.jaxa.jp” からダウンロード．       
* **取得手順2**:
    1.  **アカウント作成**: G-Portalにてユーザー登録を行い，SFTPアクセス用のパスワードを設定．
    2.  **FTPsoft**: 任意のFTPソフトを用いてアクセスする．
    3.  **データ取得**: 
* **ファイル形式**: NetCDF4，HDF5

### ひまわり (Himawari-8/9 / AHI)
気象庁が運用し，JAXAが研究用プロダクト（P-Tree）として提供している静止気象衛星．

* **プラットフォーム**: [JAXA Himawari Monitor (P-Tree)](https://www.eorc.jaxa.jp/ptree/index_j.html)
* **取得データ**:
    * L2 SST 準リアルタイム (Near Real-Time)
    * L3 SST 日平均再解析 (Analysis)
* **取得手順**:
    1.  **利用申請**: P-Tree専用フォームから申請し，FTP接続情報を取得．
    2.  **サーバー接続**: **FTPクライアント**を用いて ”ftp.ptree.jaxa.jp” に接続．
    3.  **ディレクトリ選別**:
        * **L2 NRT**: ”/pub/himawari/L2/SST/10/”（10分毎のフォルダ）から取得．
        * **L3 Analysis**: ”/pub/himawari/L3/SST/”（再解析データのフォルダ）から取得．
    4.  **一括取得**: 日付・時間ごとに整理されたディレクトリから必要なファイルを転送．
* **ファイル形式**: NetCDF4

## 2. データスペック比較

研究で使用している各プロダクトの特性を詳細に比較します．

| 項目 | GCOM-C (L2 NRT) | ひまわり (L2 NRT) | ひまわり (L3 Analysis) |
| :--- | :--- | :--- | :--- |
| **空間解像度** | **250m** | 2km | 2km |
| **時間解像度** | 約12時間に1回 | **10分** | 1日 (日平均) |
| **観測範囲** | 日本近海 (タイル単位) | 日本付近 (観測領域固定) | 日本付近 (格子状) |
| **データの性質** | 高精細・速報値 | 高頻度・速報値 | **高精度・統計処理済** |
| **欠損の性質** | 雲の影響を強く受ける | 頻度が高い分，雲の隙間を補完可能 | 統計処理によりノイズ・欠損が軽減 |
